## 1. 全局技术约束落实（铁律 → 具体做法）

| 铁律 | 落实方式 |
|------|---------|
| 金额/价格一律 `Decimal`，禁止 float | SQLite **没有原生 Decimal**，`Numeric` 在 SQLite 默认走 float 会丢精度。**必须自定义 `TypeDecorator` 以 TEXT 存储 Decimal**（见 §3.1 `Money` 类型），保证对账金额差=0。Pydantic schema 字段用 `Decimal`，`model_config` 允许 `Decimal`，序列化为字符串避免 JS 浮点污染。 |
| 证券代码字符串 | `stock_code: Mapped[str] = mapped_column(String(10))`，禁止 int；导入解析按字符串保前导零。 |
| SQLite WAL | `db.py` 用 `event.listens_for(Engine,"connect")` 执行 `PRAGMA journal_mode=WAL; PRAGMA foreign_keys=ON; PRAGMA synchronous=NORMAL;`；`create_engine(..., connect_args={"check_same_thread": False})`。 |
| Alembic 第一天接入 | M1 即 `alembic init`，所有建表走迁移，**禁止 `Base.metadata.create_all` 作为正式建表**（可仅用于测试内存库）。 |
| SQLAlchemy 2.0 风格 | 全部 `Mapped[]` / `mapped_column`，`DeclarativeBase`。 |
| 外部源统一封装 + 重试 + fallback | 所有行情走 `services/market.py` 的 `MarketDataProvider`，业务层不直接 `import tushare/akshare`；带 `tenacity` 风格重试（可手写）与 TuShare→akshare fallback。 |

## 3. 数据模型与迁移（M1 完成建表）

### 3.1 自定义 Decimal 类型（`models/types.py`）
```python
class Money(TypeDecorator):
    impl = String
    cache_ok = True
    def __init__(self, scale=4): ...
    def process_bind_param(self, value, dialect):   # Decimal -> 规整字符串
        return None if value is None else str(Decimal(value).quantize(Decimal(10)**-self.scale))
    def process_result_value(self, value, dialect): # 字符串 -> Decimal
        return None if value is None else Decimal(value)
```
所有金额/价格列用 `Money(4)`。

### 3.2 表清单（均走 Alembic 迁移）

- **import_batches**：id, filename, file_hash(unique), period_start/end(date), row_count(int), imported_at(datetime)。
- **raw_import_rows**：id, import_batch_id(FK), row_no(int), raw_text(str), parsed(bool), error(str|null)。
- **trades**（单一事实源）：id, trade_date(date), seq(int), ts_code(str), stock_code(str10), stock_name(str), side(enum buy/sell/transfer_in), price(Money), quantity(int), amount(Money 权威), fee(Money 默认0), market(enum SZ/SH), source(str), remark(str|null), raw_row_id(FK|null), import_batch_id(FK), created_at。
  - **唯一约束**：`UniqueConstraint(trade_date, stock_code, side, price, quantity, amount)`。
  - 索引：`(stock_code)`、`(trade_date)`。
- **trade_intents**：id, trade_id(FK|null), stock_code(str|null), tags(JSON 字符串数组), thesis(text|null), confidence(int|null 1-5), created_at, updated_at。
- **tags**（标签主表，供标签管理增删改）：id, name(unique), color(str|null), created_at。意图侧仍以 JSON 存名字数组；重命名标签时 service 内同步刷新 intents.tags（§7）。
- **review_reports**：id, scope(enum trade/stock/period), trade_id(FK|null), stock_code(str|null), period_start/end(date|null), content(text), model(str), rule_version_id(FK|null), input_snapshot(JSON), created_at。
- **rule_versions**：id, content(text), summary(str|null), created_at。"当前规则"= 最新一条；还原=以旧内容新建一条。
- **app_settings**：key(PK str), value(JSON/str)。存 `ollama_base_url`、`ollama_model`。
- **daily_prices**（行情缓存，净值曲线/昨收用）：id, ts_code, trade_date(date), close(Money), pre_close(Money|null)，`UniqueConstraint(ts_code, trade_date)`。
- **index_daily_cache**：id, index_code(str 如 000300.SH/000001.SH), trade_date, close(Money)，unique(index_code, trade_date)。

> enum 用 SQLAlchemy `Enum` 或 `String`+校验；SQLite 下 `String`+Pydantic Literal 更省事，推荐字符串存值。

## 4. 手续费规则（`services/fee.py`，PRD 2.8 / Q2）

输入：`side, market, amount(Decimal)` → 返回 `fee(Decimal)`，全程 Decimal、`ROUND_HALF_UP` 量化到 2 位（金额）：
- 佣金 = `max(amount * 0.00008, 5.00)`，buy/sell 双向。
- 印花税 = `amount * 0.0005`，**仅 sell**。
- 过户费 = `amount * 0.00001`，**仅沪 A（market=SH）双向**；深 A 不单算。
- `transfer_in`：fee=0，不计任何费。
- `fee = 佣金 + 印花税 + 过户费`。

导入入库时即算好写入 `trades.fee`。

## 5. 导入服务（M2，`services/importer.py`，功能 2 / PRD 2.1–2.8）

流程：
1. 读 bytes → `file_hash = sha256`。命中已有 `import_batches.file_hash` → 拒绝，返回"已导入"（PRD 2.3）。
2. **格式探测**：内容含 `<table>`/`<html>`（大小写不敏感）→ `pandas.read_html()` 兜底（PRD 2.7）；否则按 **GB18030 解码 + `\r\n` 分行 + `\t` 分列**，剔除行尾空尾列（技术方案 §5）。
3. 表头校验：匹配 `成交日期/证券代码/证券名称/买卖标志/成交价格/成交数量/成交金额/交易市场/摘要`。
4. **逐行解析**（失败行不阻断，收集 `{row_no, raw_text, error}`，PRD 2.5）：
   - 日期 `YYYYMMDD`→date；代码按 str 保前导零；
   - `买卖标志`：证券买入→buy / 证券卖出→sell / **担保品划入→transfer_in**（PRD 2.2，不计买入资金）；其余值→报错行；
   - price/amount 用 `Decimal`，**以成交金额为权威**；
   - 市场全角归一：`深Ａ→SZ` `沪Ａ→SH`；`ts_code`：SZ→`代码.SZ`、SH→`代码.SH`；
   - `seq` = 文件内行序（同日多笔相对次序）；`fee` 由 §4 计算；`source="ths_xls"`。
5. **逐行去重**：靠唯一约束；插入捕获 `IntegrityError`→该行计入 skipped（PRD 2.4）。建议每行 savepoint 或先内存查重再批量插，避免一行失败回滚整批。
6. 写 `raw_import_rows`（含 parsed/error）+ `trades` + `import_batches`（period 从文件名 `YYYYMMDD_YYYYMMDD` 解析，失败则取 trades 的 min/max trade_date）。
7. 返回摘要：`{batch_id, inserted, skipped_dup, failed:[...]}`。

## 6. 盈亏 / 持仓 / 净值（M3，`services/pnl.py`，功能 1）

### 6.1 FIFO 引擎（Q1）
- 按 `(trade_date, seq)` 排序遍历每只票的 trades。
- `buy`/`transfer_in`：压入买入批次队列 `lot(qty, price, fee_per_share, is_transfer)`；`transfer_in` 成本按划入价、**不计入"买入资金投入"统计**、自身不产生已实现盈亏。
- `sell`：从队首消耗批次配对，产出 round-trip：
  `净已实现 = 卖出金额 − 配对买入成本 − 配对买入费(按消耗比例分摊) − 本次卖出费(按本次卖量分摊)`（PRD 1.2，全 Decimal，对账差=0）。
- 卖空/数量不足：记为异常但不崩（A 股正常无卖空，作为数据校验告警）。

### 6.2 持仓总览（功能 1.1/1.3/1.4，PRD 6.2）
对剩余批次：持仓数量、持仓均价(剩余批次加权)、最新价(§8 行情)、市值、浮动盈亏=qty×(最新−均价) 及盈亏率、当日盈亏=qty×(最新−昨收)。

### 6.3 净值曲线 + 绩效摘要（功能 0.1/0.2/0.3）
- 取首笔成交日→今天的交易日历（`market.trade_cal`）。
- 对每个交易日：用"截至当日累计 trades"得当日各票持仓数量 × 当日收盘价（`daily_prices` 缓存，缺则拉取并缓存）= 当日账户市值；首日归一化 1.0。
- 叠加指数：`index_daily_cache`（000300.SH 沪深300；可切 000001.SH 上证）同期归一化。
- 绩效摘要：总收益率、最大回撤（净值序列峰谷）、累计已实现盈亏（Σ round-trip 净）、当前浮动盈亏。
- 性能：个人持仓票数少、天数有限；批量拉取 + 缓存即可，注意 TuShare 50次/分钟限频（加节流/批量）。

## 8. 行情服务（M3，`services/market.py` + `sentiment.py`，功能 1.6 / 0.4 / 6.1）

`MarketDataProvider` 统一接口（TuShare 主、akshare 兜底、重试）：
- `get_latest_price(ts_codes)`→`{code:{price, pre_close}}`：盘中最新价用 akshare 实时（如 `stock_zh_a_spot_em`），收盘后可用 TuShare `daily` 最新一日；两者皆失败→上层提示"行情获取失败"不崩（PRD 1.6）。
- `get_daily(ts_code, start, end)` / `get_index_daily(index_code,...)`：写入缓存表，缺补。
- `trade_cal(start,end)`：交易日历（TuShare `trade_cal`，akshare 兜底）。
- **大盘情绪**（`sentiment.py`，akshare）：全市场涨/跌/平家数与比例、涨停/跌停家数、成交额（亿元）、数据日期/交易状态；按涨跌比打标签（>1.5 偏多🟢 / 0.7–1.5 中性⚪ / <0.7 偏空🔴）；非交易日提示休市。

> 注意 TuShare `ts_code` 后缀已在导入时生成；指数代码沪深300=`000300.SH`、上证=`000001.SH`。
