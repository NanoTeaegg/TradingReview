## 1. 整体布局与导航

### 1.1 页面框架

```
┌─────────────┬──────────────────────────────────────────────┐
│             │                                              │
│  顶部双行导航（固定）                                     │
├────────────────────────────────────────────────────────────┤
│  主内容区                                                  │
│             │                                              │
└─────────────┴──────────────────────────────────────────────┘
```

- 顶部导航固定在页面顶部，分两行：第一行放品牌与全局操作，第二行放主导航
- 第一行内容宽度大于第二行；两行之间使用细分割线，顶部与页面背景保持一致
- 主内容区独立纵向滚动
- 右侧滑出面板（意图录入）覆盖主内容区右侧，从顶部导航栏下方开始

### 1.2 导航项结构

| 顺序 | 导航项 | 路由 | 说明 |
|------|--------|------|------|
| 1 | 交易总览 | `/` | 系统默认首页；绩效摘要 + 净值图 + 成交流水 |
| 2 | 当日持仓 | `/holdings` | 盘面解析 + 持仓明细 + 明日操作思路 |
| 3 | 交易意图 | `/intents` | 意图列表 + 复盘统计看板（Tab 切换）|
| 4 | 复盘报告 | `/reviews` | AI 报告历史列表 + 时间段复盘入口 |
| 5 | 交易规则 | `/rules` | 个人交易系统规则编辑（独立顶层页）|
| 全局操作 | 设置 | `/settings` | 位于顶部第一行右侧 |
| 全局操作 | 上传文件 | — | 位于顶部第一行右侧，点击弹窗上传 |
| 全局操作 | 导入历史 | — | 位于顶部第一行右侧，点击弹窗查看导入记录 |

> **已移除的独立页面：** 盈亏页（`/pnl`）已重命名为当日持仓（`/holdings`）并完全重写；流水页（`/trades`）已删除，成交流水并入交易总览底部。

**当前页高亮规则：** 当前激活导航项使用底部下划线；子页面（如复盘详情页）不出现在导航中，通过面包屑显示层级。

### 1.3 全局交互规范

**加载状态：** 页面级数据加载时，主内容区显示骨架屏（Skeleton）而非 loading spinner，维持布局稳定，最长等待 3 秒。

**错误提示：** 操作性错误（保存失败、行情获取失败）用页面右上角 Toast 提示，3 秒后自动消失，不打断用户操作流；致命错误（数据库连接失败）用内联 Banner 横幅展示在主内容顶部，需用户主动关闭。

**不可逆操作确认：** 删除标签、清除历史等操作触发确认 Dialog，标题说明操作后果，确认按钮标红，无倒计时。

## 2. 交易总览页 (Dashboard `/`)

### 2.1 首次使用空状态（无任何导入数据）

**触发条件：** `trades` 表为空，或系统从未完成过导入。

**页面布局：**

```
┌──────────────────────────────────────────────────────┐
│  绩效摘要卡片（4个）          ← 显示 "—" 占位符     │
├──────────────────────────────────────────────────────┤
│  净值曲线区域                  ← 空状态提示文字:     │
│                                  "导入成交后显示"    │
├──────────────────────────────────────────────────────┤
│  成交流水区块                  ← 显示"暂无成交记录"  │
└──────────────────────────────────────────────────────┘
```

**验收标准：**
- 绩效摘要四个指标值均显示 `—`，无数值计算
- 净值曲线区域中央显示文字"导入成交后显示净值曲线"
- 成交流水区块显示"暂无成交记录"空态文字

### 2.2 正常状态（已有数据）

**版块布局（上至下）：**

1. **绩效摘要卡片行**（4 个平铺卡片）
   - 总收益率、最大回撤、累计已实现盈亏、当前浮动盈亏
   - 盈亏类数值：正数绿色 + 上箭头，负数红色 + 下箭头

2. **账户净值 vs 大盘对比图**
   - 折线图，双线：账户净值曲线（主色）+ 沪深300（辅色）
   - 起点均归一化为 1.0
   - 右上角切换按钮：「沪深300」/「上证综指」（默认沪深300）
   - 横轴为交易日，纵轴为归一化净值
   - 悬停显示 Tooltip：当日账户净值、指数值、涨跌幅

3. **成交流水区块**
   - 区块标题「成交流水」
   - 筛选栏：股票代码/名称输入框 + 买卖方向下拉（全部/买入/卖出/担保品划入）
   - 表格列：日期、股票、方向、成交价、数量、成交金额、意图标签、操作（编辑意图 + AI 复盘）
   - 右侧滑出面板「编辑交易意图」（见 § 4 交易意图录入）

## 3. 当日持仓页 (`/holdings`)

### 3.1 页面布局（上至下）

```
┌──────────────────────────────────────────────────────┐
│  盘面解析（三栏卡片）                                 │
│  市场情绪 | 涨跌停统计 | 资金概况                    │
├──────────────────────────────────────────────────────┤
│  持仓明细                                            │
│  汇总行：总持仓市值 / 总浮动盈亏 / 股票数            │
│  持仓表格                                            │
├──────────────────────────────────────────────────────┤
│  明日操作思路（文本框）                               │
└──────────────────────────────────────────────────────┘
```

### 3.2 盘面解析区块

三栏平铺卡片，数据来源 akshare：

| 卡片 | 内容 |
|------|------|
| 市场情绪 | 上涨/下跌/平盘家数；情绪标签（偏多/中性/偏空）|
| 涨跌停统计 | 涨停家数、跌停家数、涨跌停比 |
| 资金概况 | 全市场成交额（亿元）、数据日期、交易状态 |

非交易日：各卡片显示「休市」提示。

### 3.3 持仓明细区块

**汇总行：** 总持仓市值、总浮动盈亏（含正负色彩）、持仓股票数。

**持仓表格：**

| 字段 | 说明 |
|------|------|
| 股票代码 + 名称 | |
| 持仓数量（股） | 整数 |
| 持仓均价（FIFO）| 元/股，Decimal 精度 |
| 最新价 | 来自行情服务 |
| 市值 | 持仓数量 × 最新价 |
| 浮动盈亏金额 | 正绿负红 |
| 浮动盈亏率 | 百分比，正绿负红 |
| 操作 | 「AI 复盘」按钮 → 跳转 `/reviews/stock/{code}` |

**排序：** 默认按市值降序；市值、浮动盈亏、盈亏率三列表头可点击切换排序。

**行情获取状态：**
- **加载中：** 最新价/市值/盈亏列显示 Skeleton 占位
- **获取失败：** 页面顶部显示黄色 Banner「行情获取失败，市值与浮动盈亏无法计算」；其他字段正常展示

**空状态：** 「暂无持仓数据。导入成交记录后，持仓将自动计算。」

### 3.4 明日操作思路区块

- 多行自由文本输入框，placeholder「记录明日计划：关注标的、止盈止损位、仓位调整思路…」
- 右上角「保存」按钮；点击后变为「已保存」绿色反馈约 1.8 秒，随后恢复
- 内容通过 localStorage 持久化；后续版本同步到服务端
- 页面重载后内容保留

## 4. 交易意图录入——右侧滑出面板

> 入口来自交易总览页成交流水区块，点击某行「编辑意图」图标触发。

**面板动画：** 从页面右边缘滑入，耗时 200ms，背景变暗（遮罩层）但仍可见轮廓；点击遮罩层或 ✕ 关闭。

**面板内容（从上至下）：**

```
┌─────────────────────────────────┐
│ ✕   编辑交易意图                │
├─────────────────────────────────┤
│ 贵州茅台 600519                 │
│ 2026-05-28  买入  ¥1,523.00     │
│ 100 股 · 成交金额 ¥152,300      │
├─────────────────────────────────┤
│ 标签（多选）                    │
│ [突破] [补仓] [止损] [+ 新建]   │
│                                 │
│ 信心度（1-5 颗星）              │
│ ☆ ☆ ☆ ☆ ☆                     │
│                                 │
│ 交易思路                        │
│ ┌─────────────────────────────┐ │
│ │（可选，不限字数）           │ │
│ └─────────────────────────────┘ │
├─────────────────────────────────┤
│  [取 消]         [保 存]        │
└─────────────────────────────────┘
```

**交互细节：**

| 操作 | 系统响应 |
|------|---------|
| 点击已有标签 | 标签切换选中/取消选中状态 |
| 点击「+ 新建」 | 行内弹出小输入框，输入标签名后回车确认；新标签自动选中并加入全局标签库 |
| 点击星星 | 填充对应颗数；再次点击同一颗星取消全部（清零）|
| 点击「保存」 | 按钮变为「保存中...」+ spinner |
| 保存成功 | 面板顶部显示「✓ 已保存」绿色提示 1.5 秒；背景列表该行意图标签同步更新 |
| 保存失败 | 面板顶部显示「保存失败，请重试」红色提示；原数据不丢失 |
| 点击「取消」或遮罩 | 若有未保存改动，弹出「有未保存的修改，确认放弃？」确认框 |

## 5. 文件上传流程（顶部导航栏触发）

### 5.1 触发方式

点击顶部第一行「上传文件」按钮，弹出文件选择框；或点击「导入历史」查看导入记录弹窗。

### 5.2 上传步骤

**步骤 1 - 选择文件**

| 操作 | 系统响应 |
|------|---------|
| 用户点击「上传文件」选择 .xls/.csv | 弹窗内显示文件名，按钮变为「解析中...」+ spinner，禁用再次上传 |
| 选择非 .xls/.csv 文件 | 弹窗内显示红色错误「仅支持 .xls 或 .csv 文件」|

**步骤 2 - 解析中（≤ 3 秒）**

弹窗显示 spinner + 文字「正在解析 文件名.xls...」；此时可继续浏览页面，导航不禁用。

**步骤 3A - 解析成功**

弹窗内展示结果摘要：
```
✅ 导入完成
成功入库：248 行    跳过重复：12 行    失败：2 行
```
绩效卡片、净值曲线自动刷新；若「失败行数 > 0」显示失败行详情。

**步骤 3B - 解析失败（文件格式错误）**

弹窗内显示红色提示：「无法解析该文件：[具体原因]」

**步骤 3C - 重复文件**

弹窗内显示黄色提示：「该文件已于 [时间] 导入过（共 248 行），已阻止重复入库。」

### 5.3 导入历史弹窗

点击顶部「导入历史」按钮弹出，每行显示：文件名、导入时间、覆盖日期区间、成功入库行数。空态显示「暂无导入记录」。

页面顶部有两个 Tab：「意图列表」和「复盘统计」。

### 6.1 Tab 1 - 意图列表

**筛选栏：**
- 股票代码输入框
- 标签多选下拉
- 日期范围选择器（按意图创建时间）

**列表每行显示：**
- 股票代码 + 名称
- 标签徽标（最多显示 3 个，超出显示「+N」）
- 思路文本前 80 字（超出省略号）
- 盈亏摘要（已实现 / 浮动，金额 + 百分比）
- 创建时间
- 「查看详情」按钮 → 跳转至 复盘详情页

**排序：** 默认按 `created_at` 倒序。

**空状态：** 「还没有意图记录。在交易总览页成交流水区块点击任意成交记录开始添加。」

### 6.2 Tab 2 - 复盘统计看板

**模块布局（上至下）：**

**模块 A - 整体胜率与盈亏比**
- 总胜率：已盈利平仓 round-trip 数 / 总平仓 round-trip 数
- 平均盈利幅度（%）/ 平均亏损幅度（%）/ 盈亏比（盈利/亏损均值之比）
- 若总平仓数 < 5，显示黄色提示「样本量不足，数据仅供参考」

**模块 B - 标签维度胜率拆分**
- 每个标签一行：标签名、交易次数、胜率、平均盈亏金额、平均持仓天数
- 默认按交易次数降序，表头可点击排序

**模块 C - 交易系统符合率**
- 圆形进度图：已打标签成交数 / 总成交数（%）
- 低于 60% 时圆形进度图变橙色，下方显示「⚠ 交易纪律执行偏低，建议及时补录意图」
- 支持按近 30 天 / 近 90 天 / 全部 切换

**模块 D - 月换手率趋势**
- 近 6 个月折线图（横轴月份，纵轴换手率 %）
- 超过 200% 的月份数据点标红 + Tooltip「⚠ 操作频繁」标注
- 数据不足 6 个月时，只显示有数据的月份，不补空月

**空状态（无任何平仓成交）：** 「暂无足够数据进行统计。完成几笔完整买卖后再来看。」

## 7. 复盘详情页（全屏）

### 7.1 触发入口

| 来源 | 复盘范围 | 路由 |
|------|---------|------|
| 交易总览成交流水「AI 复盘」按钮 | 单笔成交 | `/reviews/trade/{trade_id}` |
| 意图列表「查看详情」按钮 | 单笔成交或无关联成交的意图 | `/reviews/trade/{trade_id}` 或 `/reviews/intent/{intent_id}` |
| 总览页持仓区块「AI 复盘」按钮 | 单只股票全程 | `/reviews/stock/{stock_code}` |
| 复盘报告页「新建时间段复盘」| 时间段 | `/reviews/period?from=&to=` |

### 7.2 页面结构（以单笔成交复盘为例）

```
面包屑：复盘报告 > 贵州茅台 600519 - 单笔成交复盘

┌──────────────────────────────────────────────────┐
│  Section 1: 交易摘要                              │
│  成交日期  买卖方向  成交价  数量  成交金额  手续费│
├──────────────────────────────────────────────────┤
│  Section 2: 盈亏信息                              │
│  已实现盈亏: ¥+2,380  (+1.56%)   [卖出已完成时]  │
│  或 当前浮动盈亏: ¥-800  (-0.52%) [仍持仓时]     │
├──────────────────────────────────────────────────┤
│  Section 3: 交易意图                              │
│  标签: [突破] [高位]   信心度: ★★★★☆            │
│  思路: "突破前高，成交量放量，计划持有至..."      │
│  [编辑意图]                                       │
├──────────────────────────────────────────────────┤
│  Section 4: AI 复盘报告                           │
│  [触发 AI 复盘]                                   │
│  ↑ 未复盘时显示按钮；已有报告时显示报告内容       │
└──────────────────────────────────────────────────┘
```

**单只股票复盘变体（`/reviews/stock/{code}`）的 Section 1：**
显示该股所有成交流水的完整列表（表格，字段同交易总览成交流水），下方汇总总盈亏。

**时间段复盘变体的 Section 1：**
显示时间段内所有成交的摘要统计，不逐行展开。

### 7.3 AI 复盘生成流程

**初始状态（未复盘）：** Section 4 显示「还没有复盘报告」+ 「触发 AI 复盘」蓝色按钮。

**步骤 1 - 触发**

| 操作 | 系统响应 |
|------|---------|
| 点击「触发 AI 复盘」| 按钮变为「生成中...」+ spinner，禁用；Section 4 出现空白 Markdown 区 |
| 后端开始流式返回 | Markdown 区文字逐字显示，模拟打字效果 |
| 生成过程中点击其他导航 | 允许离开，生成在后台继续；返回此页后内容继续填充 |

**步骤 2 - 完成**

| 状态 | 系统响应 |
|------|---------|
| 生成完成 | 按钮变为「重新复盘」（可再次触发）；右上角显示「已保存 · 2026-05-29 15:30」时间戳 |
| 报告自动存入 `review_reports` | 无需手动保存；页面右上角 Toast「复盘报告已保存」|

**步骤 3 - 异常**

| 异常 | 系统响应 |
|------|---------|
| Ollama 未启动 / 连接失败 | Section 4 显示红色 Banner「Ollama 连接失败，请检查设置页的 Ollama 地址配置」+ 「去设置」跳转链接；按钮恢复可点击 |
| 生成超时（60 秒无响应）| 显示「AI 响应超时，请重试」Toast；已生成内容不丢失（如有部分内容则保留）|
| 模型不存在 | 显示「模型 {model_name} 不存在，请在设置页修改」Toast |

**验收标准：**
- 点击触发按钮到首个字符出现 ≤ 5 秒（Ollama 本地响应）
- 生成过程中可正常使用页面其他区域（Section 1/2/3 不锁定）
- 报告完成后刷新页面，内容不丢失（已持久化）
- 重新复盘时不覆盖历史报告，追加新版本（可在复盘报告页查看所有历史版本）

## 9. 交易规则页

### 9.1 主编辑区

**布局：**

```
┌────────────────┬──────────────────────────────────────┐
```

## 10. 设置页

### 10.1 Ollama 配置

**表单字段：**
- Ollama Base URL（默认 `http://localhost:11434`）
- 模型名称（默认 `qwen2.5:14b`）
- 「测试连接」按钮

**测试连接流程：**

| 操作 | 系统响应 |
|------|---------|
| 点击「测试连接」| 按钮变为「测试中...」|
| 连接成功 | 绿色行内提示「✓ 连接成功，模型可用」|
| 连接失败（URL 错误）| 红色行内提示「✗ 无法连接 Ollama，请检查地址」|
| 模型不存在 | 红色行内提示「✗ Ollama 可连接，但模型 {name} 未找到。请先 ollama pull {name}」|

**保存：** 修改任意字段后「保存」按钮激活；保存后全局复盘功能使用新配置。

### 10.2 标签管理

**列表显示：** 所有标签 + 已关联意图数。

**操作：**
- 「+ 新增标签」输入框
- 每行「重命名」inline 编辑
- 「删除」按钮：若该标签已关联意图，删除前弹出「该标签已用于 N 条意图，删除后不可恢复，确认删除？」；若未关联，直接删除

## 13. 边界与非目标

- **不涉及视觉规范**（颜色方案、字体、间距）→ ui-design skill
- **不涉及组件库选型或前端代码实现** → dev-kickoff / frontend-dev skill
- **不涉及数据库字段细节** → 见技术方案文档
- **不涉及 AI Prompt 设计**（注入规则的具体格式）→ 后端实现阶段
- **本文档不涵盖移动端交互**（PRD 明确排除）
- **本文档不定义邮件通知、推送文案**（PRD 明确非目标）
