# 产品需求文档(PRD):持仓股管理与交易复盘工具

> 状态:迭代中 · 最后更新:2026-05-30 · 负责人:AllenMakerZ

---

## 1. 背景与问题

- **要解决的问题:**  
  A 股个人投资者在做交易复盘时，面临三个割裂问题：
  1. **成交数据分散**——只有券商 App 里的原始流水，没有统一的持仓成本 / 盈亏汇总视图；
  2. **交易思路无法追溯**——下单时的判断依据（逻辑、信心度、标签）没有地方记录，事后无从复盘；
  3. **手工整理效率低**——从同花顺导出 .xls 后，还要手动在 Excel 里做 FIFO 成本配对，费时且易出错。

- **为什么现在做:**  
  持仓规模已形成、成交流水样本已具备（`20260421_20260528_Atrading.xls`），是从"凭感觉"升级到"数据驱动复盘"的合适时机。若继续推迟，历史交易意图将无法补录，复盘价值折损。

---

## 2. 目标与成功指标

- **目标:**  
  建立一个以"真实成交 + 交易意图 + 大模型复盘"为核心的个人 A 股复盘闭环；本期先把数据底座跑通，后期接入 AI 分析。

- **成功指标:**

  | 指标 | 目标值 |
  |------|--------|
  | 导入同花顺 .xls 文件并完成入库 | 成功率 ≥ 99%（非格式异常行） |
  | FIFO 盈亏计算与手工对账误差 | 金额差 = 0（Decimal 精度） |
  | 添加一笔交易意图标签耗时 | < 30 秒 |
  | 同一文件重复导入 | 系统自动拦截，不产生重复数据 |
  | 页面数据刷新（含行情） | < 3 秒内展示结果 |

---

## 3. 目标用户与场景

- **主要用户 / 角色:**  
  个人 A 股投资者（单用户工具，无多人协作需求）。

- **核心使用场景:**

  | 场景 | 触发时机 | 期望操作路径 |
  |------|----------|-------------|
  | **查看整体账户状态** | 每日开盘前/收盘后 | 打开总览页 → 一眼看到净值曲线、绩效摘要、当日大盘情绪 |
  | **导入成交** | 定期从同花顺导出流水 | 总览页拖拽上传 .xls → 解析完成 → 当页显示结果摘要 |
  | **查看持仓** | 任意时刻想了解当前持仓 | 打开总览页 → 滚动到底部持仓区块 → 看到各票持仓数量、均价、当前市值、浮动盈亏 |
  | **查看盈亏** | 阶段性复盘 | 打开盈亏页 → 看到已实现 / 未实现 / 当日盈亏汇总及明细 |
  | **记录交易意图** | 买卖后即时记录或事后补录 | 找到对应成交 → 填写思路文本、选标签、打信心度 |
  | **（未来）触发 AI 复盘** | 想对某笔或某票做深度复盘 | 选定范围 → 大模型生成分析报告（功能 4，本期预留） |

### 4.1 交易导入模块（功能 2）

> UI 入口在总览页（4.0 需求 0.5/0.6），本节定义后端解析与入库逻辑。

| 编号 | 子功能 | 用户故事 | 优先级 | 验收标准 |
|------|--------|----------|--------|----------|
| 2.1 | 上传同花顺导出文件 | 作为投资者，我想上传 .xls/.csv 文件，以便把成交数据自动入库，不再手工录入 | P0 | 上传 `20260421_20260528_Atrading.xls`（GB18030 + Tab 分隔）后，所有可解析行进入 `trades` 表；字段齐全 |
| 2.2 | 担保品划入单独识别 | 作为投资者，我想让系统区分"担保品划入"与"买入"，以便成本计算不被虚增 | P0 | `买卖标志=担保品划入` 的行映射为 `side=transfer_in`，不计入买入资金投入统计 |
| 2.3 | 重复导入拦截 | 作为投资者，我想防止同一文件重复导入，以便数据库不出现重复流水 | P0 | 同一 `file_hash` 二次上传时，系统提示"已导入"并拒绝入库 |
| 2.4 | 逐行去重 | 作为投资者，我想让系统自动忽略重复成交行，以便多次上传时不产生重复记录 | P0 | `(trade_date, stock_code, side, price, quantity, amount)` 唯一约束；重复行跳过并在结果中标注 |
| 2.5 | 错误行提示 | 作为投资者，我想知道哪些行解析失败及原因，以便我能手动处理异常数据 | P0 | 返回失败行列表（行号 + 原始文本 + 错误原因）；失败行不阻断其余行入库 |
| 2.6 | 原始行留存 | 作为投资者，我想保留原始导入文本，以便日后重新解析或追溯 | P1 | 每行原始文本存入 `raw_import_rows`，可通过 UI 或 API 查询 |
| 2.7 | HTML 伪装 .xls 兼容 | 作为投资者，我想导入其他券商的 .xls，以便工具不只支持同花顺 | P2 | 检测到 `<table>` / `<html>` 标记时自动切换 `pandas.read_html()` 解析 |
| 2.8 | 手续费自动计算 | 作为投资者，我想让系统按规则自动算出每笔手续费，以便盈亏扣费后的净收益更准确 | P1 | 按当前账本的手续费设置计算 `fee` 字段并落库：① **总佣金** = 成交金额 × 当前账本总佣金费率，买卖双向，默认万4，费率上限千3；总佣金口径对齐券商交割单「佣金」一栏，**已包含券商净佣金 + A 股固定规费（证管费 0.002% + 交易经手费 0.00341% + 过户费 0.001%），三项规费不再单独叠加**；若未勾选「免5」，总佣金最低 5 元；勾选「免5」后不启用最低额；② **印花税** = 成交金额 × 0.0005，仅卖出方向，单独计收、不计入总佣金最低额基数；`transfer_in` 行不计费 |
| 2.9 | 券商导入模板适配 | 作为投资者，我想让不同券商/不同导出版本都有明确导入模板，以便系统不会把模糊表头猜错 | P0 | 系统以 adapter 固定适配已支持模板；未知模板返回检测到的表头和缺失字段，不入库；`价格/数量/金额` 等模糊表头不能自动映射为成交字段 |

### 4.2 持仓与盈亏分析模块（功能 1）

| 编号 | 子功能 | 用户故事 | 优先级 | 验收标准 |
|------|--------|----------|--------|----------|
| 1.1 | 实时持仓总览 | 作为投资者，我想看到当前所有持仓股的数量、持仓均价、最新价、市值，以便掌握资产全貌 | P0 | 「当日持仓」页面持仓明细区块展示：股票代码、名称、持仓数量、持仓均价、最新价（行情服务）、市值、持仓盈亏金额 & 百分比；保留单票 AI 复盘入口 |
| 1.2 | FIFO 已实现盈亏 | 作为投资者，我想按先进先出算出每笔卖出的已实现盈亏（含手续费），以便准确评估历史净收益 | P0 | 卖出时与最早买入批次配对；净盈亏 = 卖出收入 − 买入成本 − 买入费用 − 卖出费用；计算结果与手工 FIFO 核对金额差 = 0。注：持仓中股票的已实现盈亏已折进持仓均价（计入持仓盈亏），「累计已实现盈亏」绩效卡仅统计已清仓股票，避免重复计算 |
| 1.3 | 持仓盈亏 | 作为投资者，我想看到当前持仓的盈亏，以便判断是否持续持有 | P0 | 持仓盈亏 = 持仓数量 × (最新价 − 持仓均价)，含该票内部已实现盈亏；最新价来自行情服务。注：因持仓均价已折入已实现盈亏，该值代表该持仓「若现在清仓」的总盈亏 |
| 1.4 | 当日盈亏 | 作为投资者，我想看到今日持仓的涨跌盈亏，以便做日内复盘 | P1 | 当日盈亏 = 持仓数量 × (最新价 − 昨收价) |
| 1.5 | 交易流水列表 | 作为投资者，我想浏览所有历史成交记录，以便追溯任意成交细节 | P0 | 交易总览页底部成交流水区块，支持按股票代码/名称、买卖方向筛选；展示所有 `trades` 表字段；不再提供独立流水页 |
| 1.6 | 行情服务（TuShare 主 + akshare 兜底） | 作为投资者，我想让系统自动获取最新行情，以便无需手动填价格 | P0 | TuShare 失败时自动 fallback 到 akshare；上层无感知；两者均失败时页面提示"行情获取失败"而非崩溃 |

### 4.4 大模型辅助复盘（功能 4）

> 支持两类大模型来源：**本地 Ollama**（无需 API Key，需自行安装并拉取模型，推荐 `qwen2.5:14b` / `deepseek-r1:7b`）与 **云端 OpenAI 兼容 API**（填厂商 Base URL + API Key + 模型名，覆盖 Deepseek / 通义千问 / Moonshot / OpenAI / 智谱等绝大多数厂商）。用户在设置页二选一并填写对应配置。

#### 大模型配置

| 编号 | 子功能 | 用户故事 | 优先级 | 验收标准 |

### 4.7 多账本（账户切换）模块

> **概念澄清**：这里的「账本」指**同一台机器、同一使用者**下的多个独立交易账本（例如：账本1、账本2、另一套交易策略），**不是**多用户登录、注册或权限体系（后者仍是非目标）。每个账本拥有独立的成交流水、出入金、意图、复盘报告与交易规则；行情数据全局共享。

## 5. 范围与非目标

**本期范围（In scope）:**
- **交易总览页面**（账户净值曲线 vs 大盘对比、绩效摘要四卡片、成交流水列表、文件上传入口与历史）
- **当日持仓页面**（盘面解析三栏、持仓明细、明日操作思路文本框）
- 功能 2 交易导入（同花顺 .xls 解析、入库、去重、错误处理）
- 功能 1 盈亏分析（FIFO 计算、持仓总览、行情接入；流水列表并入交易总览）
- 功能 3 交易意图（意图录入、关联流水+P&L 视图、标签管理、复盘统计看板）
- 功能 4 大模型复盘（本地 Ollama、单笔/单票/时间段三种触发、报告持久化）
- **交易系统规则**（独立顶层页面、Markdown 编辑、版本历史、与复盘联动）

**明确不做（Non-goals）:**
- 多用户登录 / 注册 / 账号鉴权 / 权限管理（多人协作）。注：本地「多账本切换」（4.7）属本期范围，与多用户登录是两回事——它仅在本机区分不同交易账本，不涉及身份认证
- 移动端（iOS / Android）原生应用
- T+0 分钟级盈亏（部分导入模板提供成交时间，但行情与估值仍以日级本地日线为主）
- A 股以外市场（港股、美股、期货）
- 持仓股新闻功能（整体删除）
- 推送通知（企业微信 / Server酱 / 邮件等）
- 数据导出（PDF / Excel 报告）
