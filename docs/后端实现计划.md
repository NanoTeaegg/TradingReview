## 1. 全局技术约束落实（铁律 → 具体做法）

| 铁律 | 落实方式 |
|------|---------|
| 金额/价格一律 `Decimal`，禁止 float | SQLite **没有原生 Decimal**，`Numeric` 在 SQLite 默认走 float 会丢精度。**必须自定义 `TypeDecorator` 以 TEXT 存储 Decimal**（见 §3.1 `Money` 类型），保证对账金额差=0。Pydantic schema 字段用 `Decimal`，`model_config` 允许 `Decimal`，序列化为字符串避免 JS 浮点污染。 |
| 证券代码字符串 | `stock_code: Mapped[str] = mapped_column(String(10))`，禁止 int；导入解析按字符串保前导零。 |
| SQLite WAL | `db.py` 用 `event.listens_for(Engine,"connect")` 执行 `PRAGMA journal_mode=WAL; PRAGMA foreign_keys=ON; PRAGMA synchronous=NORMAL;`；`create_engine(..., connect_args={"check_same_thread": False})`。 |
| Alembic 第一天接入 | M1 即 `alembic init`，所有建表走迁移，**禁止 `Base.metadata.create_all` 作为正式建表**（可仅用于测试内存库）。 |
| SQLAlchemy 2.0 风格 | 全部 `Mapped[]` / `mapped_column`，`DeclarativeBase`。 |
| 外部源统一封装 + 重试 + fallback | 所有行情走 `services/market.py` 的 `MarketDataProvider`，业务层不直接 `import tushare/akshare`；带 `tenacity` 风格重试（可手写）与 TuShare→akshare fallback。 |

## 3. 数据模型与迁移（M1 完成建表）

### 3.1 自定义 Decimal 类型（`models/types.py`）
```python
class Money(TypeDecorator):
    impl = String
    cache_ok = True
    def __init__(self, scale=4): ...
    def process_bind_param(self, value, dialect):   # Decimal -> 规整字符串
        return None if value is None else str(Decimal(value).quantize(Decimal(10)**-self.scale))
    def process_result_value(self, value, dialect): # 字符串 -> Decimal
        return None if value is None else Decimal(value)
```
所有金额/价格列用 `Money(4)`。

### 3.2 表清单（均走 Alembic 迁移）

> **多账本**（§15）：`accounts` 为账本主表；`trades`/`import_batches`/`cash_flows`/`trade_intents`/`review_reports`/`rule_versions`/`tags` 均新增 `account_id`(FK accounts, NOT NULL, index)；下列唯一约束改为「账本内」复合。行情/日历/`app_settings` 全局共享，不加 `account_id`。

- **accounts**：id, name(str50), kind(`live`/`demo`), is_default(bool), sort_order(int), created_at。首启无账本则建默认「主账户」。
- **import_batches**：id, account_id(FK), filename, file_hash, period_start/end(date), row_count(int), imported_at(datetime)。唯一约束 `UniqueConstraint(account_id, file_hash)`。
- **raw_import_rows**：id, import_batch_id(FK), row_no(int), raw_text(str), parsed(bool), error(str|null)。
- **trades**（单一事实源）：id, trade_date(date), seq(int), ts_code(str), stock_code(str10), stock_name(str), side(enum buy/sell/transfer_in), price(Money), quantity(int), amount(Money 权威), fee(Money 默认0), market(enum SZ/SH), source(str), remark(str|null), raw_row_id(FK|null), import_batch_id(FK), created_at。
  - **唯一约束**：`UniqueConstraint(trade_date, stock_code, side, price, quantity, amount)`。
  - 索引：`(stock_code)`、`(trade_date)`。
- **trade_intents**：id, trade_id(FK|null), stock_code(str|null), tags(JSON 字符串数组), thesis(text|null), confidence(int|null 1-5), created_at, updated_at。
- **tags**（标签主表，供标签管理增删改）：id, name(unique), color(str|null), created_at。意图侧仍以 JSON 存名字数组；重命名标签时 service 内同步刷新 intents.tags（§7）。
- **review_reports**：id, scope(enum trade/stock/period), trade_id(FK|null), stock_code(str|null), period_start/end(date|null), content(text), model(str), rule_version_id(FK|null), input_snapshot(JSON), created_at。
- **rule_versions**：id, content(text), summary(str|null), created_at。"当前规则"= 最新一条；还原=以旧内容新建一条。
- **app_settings**：key(PK str), value(JSON/str)。存大模型配置 `llm_provider`/`llm_base_url`/`llm_model`/`llm_api_key`（兼容旧键 `ollama_base_url`/`ollama_model`，见 §16）。
- **market_daily_bars**（股票/指数统一日线缓存）：id, instrument_type(`stock`/`index`), ts_code, trade_date(date), open/high/low(Money|null), close(Money), pre_close(Money|null), source, fetched_at，`UniqueConstraint(instrument_type, ts_code, trade_date)`。净值曲线、持仓估值、指数对比统一读取此表。
- **market_sentiment_snapshots**（盘面情绪快照）：id, trade_date(unique), 涨跌家数/涨跌停/成交额/情绪标签/source/note/fetched_at。盘面广度数据与日线价格语义不同，独立存储。

> enum 用 SQLAlchemy `Enum` 或 `String`+校验；SQLite 下 `String`+Pydantic Literal 更省事，推荐字符串存值。

## 4. 手续费规则（`services/fee.py`，PRD 2.8 / Q2）

输入：`side, market, amount(Decimal)` → 返回 `fee(Decimal)`，全程 Decimal、`ROUND_HALF_UP` 量化到 2 位（金额）：
- 佣金 = `max(amount * 0.00008, 5.00)`，buy/sell 双向。
- 印花税 = `amount * 0.0005`，**仅 sell**。
- 过户费 = `amount * 0.00001`，**仅沪 A（market=SH）双向**；深 A 不单算。
- `transfer_in`：fee=0，不计任何费。
- `fee = 佣金 + 印花税 + 过户费`。

导入入库时即算好写入 `trades.fee`。

## 5. 导入服务（M2，`services/importer.py`，功能 2 / PRD 2.1–2.8）

流程：
1. 读 bytes → `file_hash = sha256`。命中已有 `import_batches.file_hash` → 拒绝，返回"已导入"（PRD 2.3）。
2. **格式探测**：内容含 `<table>`/`<html>`（大小写不敏感）→ `pandas.read_html()` 兜底（PRD 2.7）；否则按 **GB18030 解码 + `\r\n` 分行 + `\t` 分列**，剔除行尾空尾列（技术方案 §5）。
3. 表头校验：匹配 `成交日期/证券代码/证券名称/买卖标志/成交价格/成交数量/成交金额/交易市场/摘要`。
4. **逐行解析**（失败行不阻断，收集 `{row_no, raw_text, error}`，PRD 2.5）：
   - 日期 `YYYYMMDD`→date；代码按 str 保前导零；
   - `买卖标志`：证券买入→buy / 证券卖出→sell / **担保品划入→transfer_in**（PRD 2.2，不计买入资金）；其余值→报错行；
   - price/amount 用 `Decimal`，**以成交金额为权威**；
   - 市场全角归一：`深Ａ→SZ` `沪Ａ→SH`；`ts_code`：SZ→`代码.SZ`、SH→`代码.SH`；
   - `seq` = 文件内行序（同日多笔相对次序）；`fee` 由 §4 计算；`source="ths_xls"`。
5. **逐行去重**：靠唯一约束；插入捕获 `IntegrityError`→该行计入 skipped（PRD 2.4）。建议每行 savepoint 或先内存查重再批量插，避免一行失败回滚整批。
6. 写 `raw_import_rows`（含 parsed/error）+ `trades` + `import_batches`（period 从文件名 `YYYYMMDD_YYYYMMDD` 解析，失败则取 trades 的 min/max trade_date）。
7. 返回摘要：`{batch_id, inserted, skipped_dup, failed:[...]}`。

## 6. 盈亏 / 持仓 / 净值（M3，`services/pnl.py`，功能 1）

### 6.1 FIFO 引擎（Q1）
- 按 `(trade_date, seq)` 排序遍历每只票的 trades。
- `buy`/`transfer_in`：压入买入批次队列 `lot(qty, price, fee_per_share, is_transfer)`；`transfer_in` 成本按划入价、**不计入"买入资金投入"统计**、自身不产生已实现盈亏。
- `sell`：从队首消耗批次配对，产出 round-trip：
  `净已实现 = 卖出金额 − 配对买入成本 − 配对买入费(按消耗比例分摊) − 本次卖出费(按本次卖量分摊)`（PRD 1.2，全 Decimal，对账差=0）。
- 卖空/数量不足：记为异常但不崩（A 股正常无卖空，作为数据校验告警）。

### 6.2 持仓总览（功能 1.1/1.3/1.4，PRD 6.2）
持仓数量(剩余批次)、持仓均价=净投入现金÷持仓股数(摊薄成本,清仓重置)、最新价(§8 行情)、市值、持仓盈亏=qty×(最新−持仓均价)(含已实现) 及盈亏率=持仓盈亏÷净投入、当日盈亏=qty×(最新−昨收)。已实现盈亏仍由 FIFO 引擎逐笔配对(§6.1)用于对账,绩效汇总「累计已实现盈亏」仅计已清仓股票。

### 6.3 出入金记录（功能 4.2）
- 新增 `cash_flows` 表：`id`、`flow_date`、`flow_type(deposit/withdraw)`、`amount`、`created_at`。
- API：`GET /api/cash-flows` 返回记录列表与累计净入金；`POST /api/cash-flows` 新增入金/出金；`DELETE /api/cash-flows/{id}` 删除记录。
- 第一条资金记录默认生效于最早成交日；后续记录默认生效于录入当日。

### 6.4 收益率走势（净值曲线）+ 绩效摘要（功能 0.1/0.2/0.3）
口径对齐券商最佳实践（富途/老虎/Moomoo）：把账户视为一只基金算净值，**默认时间加权（TWR）**，不依赖 FIFO 成本配对。
- 取首笔成交日→今天的交易日历（`market.trade_cal`）。
- 当日账户总资产 = 累计净入金 + 自有现金变动 + 自有持仓市值。
  - 持仓数量按累计买卖得到（与买卖顺序无关，无需 FIFO）；持仓成本展示用摊薄成本价（§6.2）。
  - **担保品划入/划出 = 无效数据，整体剔除**：不计入本金、不计入持仓市值；卖出担保品的现金收入同样剔除（FIFO 命中划入批次的部分不计入自有现金，避免凭空多出钱）。
  - 融资账户允许买入额 > 本金（持仓市值可大于净入金），属正常，不影响以本金为基准的收益率。
- 时间加权净值 NAV：起点 1.0；每日 `r_t =（今日总资产 − 昨日总资产 − 当日出入金净流入）/（昨日总资产 + 当日入金）`，`NAV_t = NAV_{t-1}×(1+r_t)`，剔除出入金扰动。单笔入金场景等价于「账户总资产 / 本金」。
- 接口 `GET /api/positions/equity-curve` 返回：`dates`、`nav`（TWR 起点 1.0）、`total_assets`（当日总资产）、`flows`（区间出入金）、`benchmarks`（沪深300/上证/深证成指/创业板指，起点归一）。前端按所选区间二次归一为 0% 起点，头部收益率与曲线均取 TWR；支持日期筛选（近1周/近1月/年初至今/全部/自定义）。
- 绩效摘要：总收益率 = TWR 末值 − 1；最大回撤基于 TWR 净值序列峰谷；总盈亏（不含手续费）= 与净值曲线同序列的账户累计盈亏（期末总资产 − 期初总资产 − 区间净出入金），与前端「累计收益」全部区间一致；当前持仓盈亏仍按摊薄成本；另列累计手续费。
- 性能：个人持仓票数少、天数有限；批量拉取 + 缓存即可，注意 TuShare 50次/分钟限频（加节流/批量）。摘要页用本地缓存行情快速估算 NAV。

## 9. 复盘（M5，功能 4 全量，`services/review.py` + `rules.py`）

  - 组装 `input_snapshot`：成交数据 + 当时/区间行情 + 关联意图(tags/thesis/confidence) + **当前规则全文(rule_versions 最新)** + 记录 `rule_version_id`（4.5/5.4 可复现）。
  - 调 Ollama `POST {base}/api/chat`（`stream=true`，用 `httpx.AsyncClient.stream`）→ FastAPI `StreamingResponse`（逐 token 透传，SSE 或 chunked）。
  - 流结束落库 `review_reports`（content + model + input_snapshot + rule_version_id）。
- **报告列表/详情**（4.6/4.7）：按 `created_at` 倒序；详情返回全文 + snapshot。
- **规则模块**（功能 4.5 PRD 5.x，`rules.py`）：`GET /api/rules`(当前)、`PUT /api/rules`(保存→新建 rule_version，附 summary)、`GET /api/rules/versions`、`POST /api/rules/versions/{id}/restore`。章节大纲(5.2)前端解析 Markdown，后端只存全文。

## 15. 多账本（账户切换）实现（PRD 4.7 / 技术方案 §6.0）

### 15.1 模型（`models/account.py`）
```python
class Account(Base):
    __tablename__ = "accounts"
    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str] = mapped_column(String(50), nullable=False)
    kind: Mapped[str] = mapped_column(String(10), default="live")   # live/demo
    is_default: Mapped[bool] = mapped_column(Boolean, default=False)
    sort_order: Mapped[int] = mapped_column(Integer, default=0)
    created_at: Mapped[datetime] = mapped_column(DateTime, server_default=func.now())
```
`trades`/`import_batches`/`cash_flows`/`trade_intents`/`review_reports`/`rule_versions`/`tags` 各加
`account_id: Mapped[int] = mapped_column(ForeignKey("accounts.id", ondelete="CASCADE"), nullable=False, index=True)`。
`raw_import_rows` 不加列（经 `import_batch_id` 间接归属）。

### 15.2 唯一约束调整
- `import_batches`：删除 `file_hash` 全局 unique，改 `UniqueConstraint("account_id", "file_hash")`。
- `trades`：唯一约束改 `("account_id", "trade_date", "stock_code", "side", "price", "quantity", "amount")`。
- `tags`：`name` 去全局 unique，改 `UniqueConstraint("account_id", "name")`。

### 15.3 迁移（新增 Alembic 版本，**禁止改历史迁移**）
1. `create_table accounts`。
2. 插入默认账本：`INSERT INTO accounts(name, kind, is_default, sort_order) VALUES ('主账户','live',1,0)`，取其 `id`（通常为 1）。
3. 对 7 张业务表：`add_column account_id`（先 nullable）→ `UPDATE 表 SET account_id = <默认id>`（`tags` 同理；存量数据全部归默认账本）→ 改 NOT NULL。
   - SQLite 不支持直接加 NOT NULL/FK，需 `batch_alter_table` 重建表并迁移约束（Alembic `op.batch_alter_table`）。
4. 在 `batch_alter_table` 内重建上述唯一约束（先 drop 旧约束/索引再建新复合约束）。
5. `downgrade` 反向：还原旧唯一约束、删列、删表。

### 15.4 账本上下文（`api/deps.py`）
- 新增依赖 `get_current_account_id`：读请求头 `X-Account-Id`（无则 query `account_id`），校验存在；缺省回退 `is_default` 账本 id。
- 所有业务 service 函数签名加 `account_id`；查询统一 `.where(Model.account_id == account_id)`，写入时落 `account_id`。
- 行情/日历/情绪/设置类接口**不**注入账本上下文（全局共享）。
- `importer`：去重查询与 `file_hash` 命中判断都加 `account_id` 过滤；同文件入不同账本视为新批次。
- `pnl`/`stats`/`positions`/`reviews`/`rules`/`intents`/`tags`/`cash_flows` 全部按 `account_id` 过滤。

### 15.5 账本 + 复制规则 API
```
GET    /api/accounts                       # 账本列表（含 is_default、kind、sort_order）
POST   /api/accounts                        # 新建 {name, kind} → 返回新账本，前端切过去
PATCH  /api/accounts/{id}                    # 重命名 / 设默认 / 调序
DELETE /api/accounts/{id}                    # 级联删业务数据；最后一个不可删（409）
POST   /api/rules/copy?from_account_id=      # 把来源账本最新规则全文写入当前账本（新建 rule_version，summary 标注「复制自 XX」）
```
其余既有路由不变，但隐式按 `X-Account-Id` 过滤。

### 15.6 演示种子（`app/seed_demo.py`，供 GitHub 演示）
- 幂等脚本：若无 `kind=demo` 账本则创建「模拟账户」，灌入虚构 `trades`/`cash_flows`/`rule_versions`（不写真实数据）。
- 真实 `*.db` 不入 Git；README 写明 `alembic upgrade head → python -m app.seed_demo →（可选）拉行情`。

## 16. 大模型多来源（PRD 4.1 / 技术方案 §9）

### 16.1 配置（`app_settings` 键）
| key | 默认 | 说明 |
|-----|------|------|
| `llm_provider` | `ollama` | `ollama` / `openai_compatible` |
| `llm_base_url` | `http://localhost:11434` | Ollama 或云端 Base（如 `https://api.deepseek.com/v1`） |
| `llm_model` | `qwen2.5:14b` | 模型名（云端如 `deepseek-chat`） |
| `llm_api_key` | 空 | 仅 `openai_compatible` 用；存本机，接口返回掩码 `sk-****1234`，**不入 Git** |

> 兼容旧键：迁移时把 `ollama_base_url`/`ollama_model` 读入新键（若存在）。

### 16.2 `LLMProvider`（`services/review.py`）
- 工厂按 `llm_provider` 返回实现：
  - `OllamaProvider`：`POST {base}/api/chat`，body `{model, messages, stream:true}`。
  - `OpenAICompatibleProvider`：`POST {base}/chat/completions`，header `Authorization: Bearer {api_key}`，body `{model, messages, stream:true}`，解析 SSE `data:` 行的 `choices[].delta.content`。
- 两者都返回异步生成器 → FastAPI `StreamingResponse`。
- 落库 `review_reports` 记 `provider`（新增列）+ `model` + `input_snapshot` + `rule_version_id`。

### 16.3 设置 API（替换原 Ollama 专用路由）
```
GET    /api/settings/llm            # 返回 provider/base_url/model + api_key 掩码
PUT    /api/settings/llm            # 保存配置
POST   /api/settings/llm/ping       # 按 provider 测试：ollama→GET {base}/api/tags；openai_compatible→GET {base}/models（带 key）或最小 chat 探测
```
> 保留 `/api/settings/ollama*` 作别名一段时间或直接迁移，前端同步改。

### 16.4 依赖
`httpx` 已在清单中，云端 `openai_compatible` 直接用 `httpx` 调 `/chat/completions`，无需新增 `openai` SDK（保持轻量）。
